Introducción

Esta documentación provee la informacion de como trabajar con nuestra API

URL Base: http://localhost

Autenticación de solicitudes

Para autenticar las solicitudes, incluye un encabezado Authorization con el valor "Bearer {YOUR_AUTH_KEY}".

Todos los endpoints autenticados están marcados con una insignia requiere autenticación en la documentación a continuación.

You can retrieve your token by visiting your dashboard and clicking Generate API token.

Authentification

Login

POST
http://localhost
/api/login

Inicia sesion de un usuario y regresa un token

Encabezados

Content-Type
Example:
application/json
Accept
Example:
application/json

Parámetros de cuerpo

email
string

El correo del usuario

Example:
gbailey@example.net
password
string

La contraseña del usuario

Example:
|]|{+-
Ejemplo de solicitud: 
const url = new URL(
    "http://localhost/api/login"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "email": "gbailey@example.net",
    "password": "|]|{+-"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());
Ejemplo de respuesta:
{ "message": "Login exitoso", "user": {"nombre": "Miguel", "id": "68b8634ccf157bf9880c1e7"}, "token": "68b898af531f622cbf0ba1", "permisos": ["usuarios_leer", "Plantillas_leer",]}
{
    "message": "Credenciales invalidas"
}
